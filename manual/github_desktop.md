# Level 2 – Using GitHub Desktop
GitHub Desktop allows you to work on project files locally using your normal software while automatically keeping your local copy synchronized with GitHub.

> [!NOTE]
> The manual for GitHub Desktop is mostly AI generated and has not been tested at the moment.
> Continue at your own risk >:)

No command-line knowledge is required.

## Before we begin..

Install:

* GitHub Desktop

Ensure that:

* You are signed in with your GitHub account.
* You have access to the IdPDE repository.

## Cloning a repository

Before you can work on a project, you need to create a local copy.

1. Open GitHub Desktop.
2. Select **Clone Repository**.
3. Choose the repository.
4. Select a location on your computer.
5. Clone the repository.

The repository now behaves like a normal folder on your computer.

## Workflow

Every working session follows the same sequence.

### Step 1 – Fetch or Pull

Before making changes:

Click **Fetch origin**.

If changes are available:

Click **Pull origin**.

This ensures you are working with the latest version.

### Step 2 – Edit files

Edit files using your preferred software.
GitHub Desktop automatically detects modified files.

### Step 3 – Review your changes

Return to GitHub Desktop.

Review:
* Which files changed
* What changed inside each file

Make sure everything you intend to save is included.

### Step 4 – Commit

Enter:

Summary:

A short description.

Optionally:

A longer explanation.

Click **Commit to main**.

A commit records your work locally.

### Step 5 – Push

After committing:

Click **Push origin**.

Your changes are now uploaded to GitHub.
